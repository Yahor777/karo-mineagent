# MineAgent Workbench — UI-спека v0.1 (Фаза 4)

Спецификация глобальной переработки интерфейса. Backend под все пункты готов
(Фазы 1–3). Реализуется во фронтенде webview: `src/webview/html.ts`,
`media/mineagent.css`, `media/mineagent.js`, `src/webview/MineAgentWebviewProvider.ts`.

## P4.1 Каркас
- Чёткий layout: шапка (проект/loader/MC-версия) → область чата → инпут → панели.
- Тёмная тема (VS Code theme variables: `--vscode-*`).
- Состояния: загрузка (skeleton), ошибка (гуманизированная — `humanizeWorkbenchError`),
  пусто (подсказка с первого действия).

## P4.2 Слэш-«/»-палитра
В поле ввода по «/» — палитра всех функций с автодополнением:
- Режимы: ask / plan / build / playtest.
- Инструменты: repo.*, build.diagnose, git.* , knowledge.search, minecraft.*.
- Индекс: структурный (repo.index), ИИ-индекс (P2.1) с прогрессом/возрастом.
- Поиск знаний, git-операции, выбор модели/effort, память (memory.note).
Каждый пункт → команда в `postMessage` к провайдеру.

## P4.3 Селекторы модели + effort, индикатор активной модели
- Дропдаун модели прямо в инпуте; отдельно — effort (low/medium/high), активен
  только для reasoning-моделей (гейт `supportsEffort`, P1.5).
- Индикатор «отвечает модель X» в шапке ответа; иконка-замок (P1.2 `lockModel`).
- Битые модели (P1.4 `annotate` → `deprecated:true`) — серым, с подсказкой.

## P4.4 Панели
- **Память**: рендер `.mineagent/project.md` (ProjectMemoryService), редактируемо.
- **База знаний**: список записей (категории/теги/источник), правка, поиск.
- **Git**: статус (`git.status`), diff, кнопки commit/branch/push (через approval).
- **Индекс**: структурный + ИИ-индекс, возраст (`indexAgeMs`), прогресс build().

## P4.5 Модалки подтверждения
- Смена модели (P1.1/P1.3): «повторить / выбрать другую / отменить» — вместо тихого фейла.
- Опасные действия (ApprovalGate, 3 кнопки): «один раз / всегда в сессии / всегда».
- Превью для git push / PR / patch перед выполнением.

## Онбординг (Фаза 5)
Мастер первого запуска: выбор провайдера → ввод ключа (SecretStorage) → выбор модели
(с preflight-пробой) → готово. Если ключа нет — баннер с кнопкой «Настроить провайдера».
